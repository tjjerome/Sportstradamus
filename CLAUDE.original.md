## Project Overview

Sportstradamus is a Python package that scrapes sportsbook odds and player stats, trains distributional ML models to predict player performance, and exports value recommendations to Google Sheets. It supports MLB, NBA, NFL, NHL, and WNBA.

## Architecture

### Data Pipeline

1. **Collection** (`books.py`): Scrapes DraftKings, FanDuel, Pinnacle, Caesars, PrizePicks, Underdog, Sleeper via their APIs. Uses `Scrape` helper with rotating headers (ScrapeOps).
2. **Enrichment** (`moneylines.py`, `stats.py`): Fetches player stats from official league APIs (mlb-statsapi, nba-api, nfl-data-py).
3. **Training** (`train.py`): Builds training matrices per market, trains LightGBMLSS distributional models, computes SHAP importances.
4. **Prediction** (`sportstradamus.py`): Scores new offers against trained models, computes EV, writes to Google Sheets via gspread.

### Core Class Hierarchy

**`Stats`** (base) → `StatsNBA`, `StatsWNBA`, `StatsMLB`, `StatsNFL`, `StatsNHL`

Each subclass implements league-specific data loading and feature engineering. Key methods:
- `load()` / `update()` — Load/fetch game logs from league APIs
- `get_training_matrix(market)` — Generate feature matrix (X) and targets (y)
- `get_stats(offer, game_date)` — Feature vector for a single prediction
- `bucket_stats(market)` / `profile_market()` — Aggregate stats for defense/offense profiling

**`Archive`** (`helpers.py`): Persistent odds storage using klepto HDF archives. Methods: `get_line()`, `get_ev()`, `write()`, `clip()`.

**`Scrape`** (`helpers.py`): HTTP client with ScrapeOps header rotation and per-book API methods (`get_dk`, `get_fd`, `get_pinnacle`, etc.).

### ML Pipeline

Uses **LightGBMLSS** for distributional regression — predicts full probability distributions, not point estimates.

Distribution types (configured in `stat_dist.json`):
- **Gamma** / **ZAGamma** — Continuous stats, with optional zero-inflation
- **Negative Binomial** / **ZINB** — Count stats, with optional zero-inflation
- **SkewNormal** (`skew_normal.py`) — Custom PyTorch distribution for LightGBMLSS

**Player Comparables**: Z-scored player profiles → weighted BallTree KNN → comp outcomes as features. Weights optimized via `scipy.optimize.differential_evolution` in `scripts/optimize_comp_weights.py`.

**Feature filtering**: SHAP-based importance thresholds stored in `feature_filter.json`, updated each training cycle.

### Training Report Diagnostics (`training_report.txt`)

Generated by `train.py:report()` after `meditate` completes. Each market block contains:

**Header**: Distribution type (NegBin/Gamma/ZINB/ZAGamma) and historical zero rate.

**Performance Table** (row 1 = raw LightGBMLSS model output, row 2 = after model correction (fit_model_weight + dispersion_cal), row 3 = after temperature scaling calibration; all filtered at confidence > 0.54):

| Metric | Meaning |
|--------|---------|
| Accuracy | `accuracy_score` on over/under classification |
| Over Prec | Precision of "Over" predictions |
| Under Prec | Precision of "Under" predictions |
| Over% | Proportion of predictions that are "Over" |
| Sharpness | Std dev of predicted probabilities (lower = more decisive) |
| NLL | Negative log-loss |

**DIAG — Model Blending & Calibration**:
- `model_weight`: Blend weight [0.05–0.9] between model and bookmaker, optimized via log-likelihood. Low = bookmaker dominates. Uses logarithmic opinion pool for NegBin, precision-weighted blend for Gamma.
- `model_calib`: `1 - mean((calibrated_prob - actual)²)` on validation set after temperature scaling. Higher = better.

**DIAG — Shape / Dispersion**:
- `start_r` / `start_alpha`: Method-of-moments shape estimate from training features (NegBin: `μ²/(σ²-μ)`, Gamma: `(μ/σ)²`).
- `model_r` / `model_alpha`: Mean shape parameter predicted by LightGBMLSS (NegBin: `total_count`, Gamma: `concentration`).
- `empirical_r` / `empirical_alpha`: Shape computed from actual test outcomes — ground truth dispersion.
- `shape_ratio`: `model_shape / empirical_shape`. >1 = over-dispersed, <1 = under-dispersed, 1.0 = ideal.
- `shape_ceiling`: Upper bound on shape parameter during training (2× `marginal_shape`).
- `marginal_shape`: Empirical shape from full training targets.
- `dispersion_cal`: Post-hoc shape scaling factor [0.1–1.0] learned on validation set. 1.0 = no correction needed, <1.0 = model was over-dispersed.

**DIAG — Expected Value**:
- `start_mean`: Average of training feature `MeanYr`.
- `model_ev`: Mean blended expected value (model + bookmaker fusion via `fused_loc()`).
- `mean_line`: Average bookmaker line.
- `result_mean`: Actual test set mean outcome.
- `mean_ev_diff` / `median_ev_diff`: How much blended EV exceeds bookmaker line (mean/median).
- `frac_ev>line`: Fraction where model EV > bookmaker line (~50% = well-calibrated).

**DIAG — Conditional Prediction Quality**:
- `Over%|ev>line`: Over prediction rate when model EV > line (should be high).
- `Over%|ev<line`: Over prediction rate when model EV < line (should be low).
- `CF_Over%(emp_shape)`: Counterfactual Over% using empirical shape instead of model shape — tests if shape miscalibration is causing mispredictions.

**HP — Hyperparameters** (from Optuna tuning):
- `rounds`: Boosting iterations | `leaves`: Max leaves per tree | `lr`: Learning rate
- `min_child`: Min samples per leaf | `L1`/`L2`: Regularization strengths

**Quick interpretation guide**:
- Sharpness low + NLL low = good confidence separation
- `Over%|ev>line` high & `Over%|ev<line` low = properly discriminative
- `shape_ratio` ≈ 1.0 = well-calibrated dispersion
- `dispersion_cal` = 1.0 = no post-hoc fix needed; <1.0 = model was over-dispersed
- `model_weight` near 0 = bookmaker dominates; near 1 = model dominates

### Key Configuration Files (in `src/sportstradamus/data/`)

| File | Purpose |
|------|---------|
| `stat_dist.json` | Distribution type per stat (Gamma, NegBin, ZINB, etc.) |
| `stat_cv.json` | Coefficient of variation per stat |
| `stat_zi.json` | Zero-inflation parameters |
| `stat_map.json` | Stat name mappings across APIs/sportsbooks |
| `feature_filter.json` | SHAP-based feature importance thresholds |
| `playerCompStats.json` | Learned player comp weights per league/position |
| `book_weights.json` | Sportsbook reliability weights for consensus lines |
| `{LEAGUE}_corr.csv` | Pre-computed player stat correlation matrices |

### Data Storage

- `data/models/` — Trained LightGBMLSS model pickles
- `data/training_data/` — Cached training matrices
- `data/player_data/{LEAGUE}/{YEAR}/` — Historical game log CSVs
- `data/test_sets/` — Holdout test data
- `creds/` — API keys (`keys.json`), Google OAuth (`credentials.json`, `token.json`)
